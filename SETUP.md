## 🚀 Quick Start — GitHub Actions Setup

### Step 1: Enable GitHub Pages
1. Go to your repo → **Settings** → **Pages**
2. Under **Build and deployment**, select:  
   **Source → GitHub Actions**
3. Click **Save**

### Step 2: Enable GitHub Actions
The workflows run automatically on push. No extra setup needed for:
- `security-review.yml` — runs on every push
- `deploy-and-test.yml` — runs on every push to main

### Step 3: Configure GitHub Pages Permissions
In your repo → **Settings** → **Actions** → **General**:
- Under "Workflow permissions", select **Read and write permissions**
- Check **Allow GitHub Actions to create and approve pull requests**

## 🖥️ Local Test Execution

### Run Selenium Tests Locally

```bash
# 1. Install test dependencies
cd selenium-tests
npm install

# 2. Copy env file
cp .env.example .env
# Edit .env: set BASE_URL=https://Viru-6281.github.io/spic-pdd/
# For local dev: set BASE_URL=http://localhost:5173/ and HEADLESS=false

# 3. Run all tests
npm test

# 4. Run specific suite
npm run test:login
npm run test:navigation
npm run test:register
```

### Generate Reports Locally

```bash
# Install Python dependencies
pip install openpyxl

# Generate security Excel reports
python scripts/generate_reports.py

# Generate test HTML report
python scripts/generate_test_report.py \
  --log "Test Results/Logs/selenium-output.log" \
  --output "Test Results/HTML/execution-report.html" \
  --base-url "https://Viru-6281.github.io/spic-pdd/"

# Generate test Excel report
python scripts/generate_test_excel.py \
  --log "Test Results/Logs/selenium-output.log" \
  --output "Test Results/Excel/Automation_Test_Report.xlsx"
```

## 📊 GitHub Actions Workflows

### Workflow 1: Security Review
**File:** `.github/workflows/security-review.yml`

| Job | Tool | Purpose |
|-----|------|---------|
| `detect-stack` | Shell | Detect Java/Node tech stack |
| `semgrep-sast` | Semgrep | SAST scan (Java + JS rules) |
| `trivy-scan` | Trivy | CVE scanning (filesystem + Maven + npm) |
| `gitleaks-scan` | Gitleaks | Secret detection |
| `owasp-dependency-check` | OWASP | Java dependency CVEs |
| `npm-audit` | npm audit | Frontend dependency audit |
| `generate-reports` | Python | Generate Excel reports |
| `security-gate` | Shell | **Fail build on Critical findings** |

> ⚠️ This workflow **intentionally fails** on Critical vulnerabilities (Wildcard CORS, No Auth, IDOR). Fix the 3 Critical findings to make it green.

### Workflow 2: Deploy + E2E
**File:** `.github/workflows/deploy-and-test.yml`

| Job | Purpose |
|-----|---------|
| `build` | Build React + inject base path for GH Pages |
| `deploy` | Deploy to GitHub Pages via Actions |
| `verify-deployment` | Wait for site + check HTTP 200 |
| `selenium-e2e` | Run 28 Selenium tests against live URL |
| `publish-reports` | Push HTML report to gh-pages/reports/ |
